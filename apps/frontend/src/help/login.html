Login guide

Choose a sign-in path and theme before entering the app.

The login page is intentionally simple: authenticate, pick a display mode if you want, and continue into your dashboard.

What this page is for

This page authenticates the user and lets you pick the visual theme before opening the rest of the application.

Important controls

  • Light, Dark, Auto: theme preference buttons.
  • Sign in with Google: the normal production login path.
  • Dev Login: a development shortcut that skips OAuth when available.

When to use which button

  1. Use Sign in with Google for normal authentication.
  2. Use Dev Login only in local development.
  3. If you already have a valid token, the page may redirect you automatically.
