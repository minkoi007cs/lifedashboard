Tasks guide

Plan work in weekly, monthly, and stats views.

The Tasks page is where you create work items, move across time, and review how much is getting done.

What this page is for

Use Tasks to track personal tasks and shared plans created from wishes. It is the project calendar layer of the app.

Important controls

  • Weekly tab: best for scheduling and seeing workload by week.
  • Monthly tab: best for scanning a larger range of upcoming items.
  • Stats tab: shows completion and performance metrics.
  • Date navigation: the arrows move the visible week or month.
  • Today button: jumps the view back to the current date.
  • New Task: opens the task form for manual task creation.

Shared plans from wishlist

When an activity wish becomes a shared plan, it also appears here. Shared plans are visually marked so you can distinguish them from normal personal tasks.

Typical workflow

  1. Create a task with title, priority, due date, and optional reminder.
  2. Use the weekly view for time-based planning.
  3. Switch to monthly when you want range and context.
  4. Check stats to see whether your system is realistic.
